# PUZZLE_QT
# Описание

Это приложение представляет собой классическую головоломку "Пятнашки" (15-puzzle), реализованную на C++ с использованием фреймворка Qt. Пользователь может выбирать изображение, которое затем разбивается на 16 частей (15 фрагментов + пустое место), которые нужно собрать в правильном порядке.

## Особенности

- Выбор пользовательского изображения для головоломки
- Адаптивный интерфейс с фоновым изображением
- Проверка на корректность решения
- Уведомление о победе
- Возможность начать новую игру или выйти из приложения

## Требования

- Qt 5 или новее
- Компилятор C++ с поддержкой C++11

## Структура проекта

- **MainWindow**: Главное окно приложения с кнопками "Новая игра" и "Выход"
- **Form**: Окно выбора изображения и управления игрой
- **Game**: Логика игры и отображение игрового поля

## Как собрать

1. Убедитесь, что у вас установлен **Qt Creator** и необходимые библиотеки Qt.
2. Откройте проект в **Qt Creator**.
3. Нажмите "Собрать" (Build).
4. Запустите приложение.

## Как играть

1. Нажмите "New game" в главном меню.
2. Выберите изображение из файловой системы (600x600 пикселей).
3. Перемещайте фрагменты, кликая по ним (можно двигать только соседние с пустым местом фрагменты).
4. Соберите изображение в исходном порядке.

## Классы

### MainWindow

- Отображает главное меню
- Обрабатывает нажатия кнопок:
    - `starT()` — начинает новую игру
    - `exiT()` — подтверждает выход из приложения

### Form

- Содержит кнопки:
    - "choose a picture" — выбор изображения для новой игры
    - "come back" — возврат в главное меню
- Управляет экземпляром игры (Game)

### Game

- Реализует логику игры:
    - `createButton()` — создает кнопки-фрагменты
    - `move()` — обработка перемещения фрагментов
    - `checkGameOver()` — проверка завершения игры
    - `openImage()/loadImage()` — загрузка пользовательского изображения
    - `setupPuzzle()` — подготовка фрагментов головоломки

## Ограничения

- Размер фрагментов фиксирован (150x150 пикселей)
- Поддерживаются только квадратные изображения (рекомендуется 600x600 пикселей)
- Интерфейс не адаптируется под разные разрешения экрана
